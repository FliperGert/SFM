# 🔬 SFM — Симулятор физических моделей

> Учебный проект по дисциплине «Языки программирования» · ИКНТ ПГНИУ · 2026  
> C++/CLI · Windows Forms · .NET Framework

---

## 📋 Описание

**SFM** (Simulator of Physical Models) — десктопное приложение для пошагового моделирования четырёх физических процессов. Пользователь вводит начальные условия, нажимает «Запуск» и получает таблицу значений по шагам времени.

Проект реализует **собственные шаблонные структуры данных** (`List<T>`, `ListModel<T>`, `Node<T>`) вместо стандартных контейнеров STL — для демонстрации принципов работы связных списков.

---

## 🧪 Реализованные модели

| # | Модель | Входные параметры | Выходные данные |
|---|--------|-------------------|-----------------|
| 1 | **Равноускоренное движение** | x₀, v₀, a, t, d | x(t), v(t), a, t |
| 2 | **Колебания пружины** (гармонический осциллятор) | k, x, m, t, d | x(t), v(t), a(t), t |
| 3 | **Баллистическое движение** (бросок под углом) | v, α, t, d | x(t), y(t), v(t), t |
| 4 | **Падение с сопротивлением воздуха** | S, k, h, m, t, d | F(t), v(t), a(t), S(t), x(t), t |

> `d` — шаг по времени (с), `t` — время симуляции (с)

### Иерархия классов

```
ModelProcess  (базовый класс — classModel.h)
├── ModelProcess_1   update(x0, v0, a, d, step)
├── ModelProcess_2   update(k, x, m, d, step)
├── ModelProcess_3   update(v, alpha, d, step, work&)
└── ModelProcess_4   update(s, k, h, m, d, S&, step, work&)

List<T>  ←  используется ModelProcess для хранения данных
└── Node<T>   (двусвязный список, list.h)

ListModel<T>  ←  хранит таблицу результатов (список строк)
└── Node<List<T>>
```

Каждая модель реализует два метода:
- `update(...)` — вычисление одного шага
- `full()` — запуск симуляции по всем шагам от `0` до `t`

## ⚙️ Формулы

<details>
<summary><b>Модель 1 — Равноускоренное движение</b></summary>

```
x(t) = x₀ + v₀·t + ½·a·t²
v(t) = v₀ + a·t
```
</details>

<details>
<summary><b>Модель 2 — Колебания пружины</b></summary>

```
ω   = √(k / m)
x(t) = x · cos(ω·t)
v(t) = −x · ω · sin(ω·t)
a(t) = −x · ω² · cos(ω·t)
```
Период колебаний: `T = 2π / ω`
</details>

<details>
<summary><b>Модель 3 — Баллистическое движение</b></summary>

```
x(t) = v · cos(α) · t
y(t) = v · sin(α) · t − ½ · g · t²    (g = 9.81 м/с²)
```
Симуляция останавливается при `y < 0` (тело достигло земли).
</details>

<details>
<summary><b>Модель 4 — Падение с сопротивлением воздуха</b></summary>

```
β       = S · k · 1.293            (ρ_воздуха = 1.293 кг/м³)
v_lim   = √(m · g / β)            (предельная скорость)
v(t)    = v_lim · tanh(g·t / v_lim)
F(t)    = β · v²
a(t)    = g − β·v² / m
```
Симуляция останавливается при достижении высоты `h`.
</details>

## 🛡️ Обработка исключений

Все формы защищены двухуровневой валидацией:

1. **Ручные проверки** перед запуском:
   - `t > 0` и `d > 0`
   - Физические параметры `m`, `k`, `S`, `h` > 0
   - `S ≠ 0` и `k ≠ 0` (защита от деления на ноль в модели 4)

2. **Блок `try-catch`**:
   - `System::FormatException` — нечисловой ввод
   - `System::Exception` — любые другие ошибки

## 📊 Структуры данных

Проект использует **собственные реализации** вместо `std::vector` / `std::list`:

```cpp
// Двусвязный узел
template<typename T>
class Node { T data; Node* next; Node* prev; };

// Односвязный список с push_back, push_front, operator[]
template<typename T>
class List { Node<T>* head; Node<T>* tail; int length; };

// Список списков (таблица результатов симуляции)
template<typename T>
class ListModel { Node<List<T>>* head; Node<List<T>>* tail; int leight; };
```

`ListModel<long double>` хранит результаты так:
```
ListModel
├── Node[ List {x=0,    v=5,  a=2, t=0} ]   ← шаг 0
├── Node[ List {x=5.1,  v=7,  a=2, t=1} ]   ← шаг 1
└── Node[ List {x=12.2, v=9,  a=2, t=2} ]   ← шаг 2
```
